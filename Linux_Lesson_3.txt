1. Создание ссылок:

- создать файл file1 и наполнить его произвольным содержимым.
cat > file1 Hello world!

- скопировать файл file1 в file2.
cp file1 file2

- создать символическую ссылку file3 на file1.
ln -s file1 file3

- создать жесткую ссылку file4 на file1.
ln file1 file4  - посмотреть, какие айноды у файлов.
ls -li
file1 и file4 имеют одинаковые айподы,  другие аноды отличаются друг от друга

- удалить file1. Что стало с остальными созданными файлами? Попробовать вывести их на экран.
rm file1

cat file2
Hello world!

cat file3
cat: file3: No such file or directory

cat file4
Hello world!

file2 и жесткая ссылка file4 без изменений, а вот по символической ссылке file3 получаем ошибку что такого файла не существует.

2. Работа со ссылками:

- дать созданным файлам другие, произвольные имена.
mv file2 file20
mv file4 file40

- создать новую символическую ссылку.
ln -s file20 sim20

- переместить ссылки в другую директорию. Работают ли ссылки? Попробовать вывести их на экран.
mv {sim20,file40} ./temp
cd /home/anastasia/temp

cat file40
Hello world!

cat sim20
cat: sim20: No such file or directory

Переименованная жесткая ссылка file40 остаётся рабочей, а символьная ссылка не существует.

 3. Права файлов:
 
 - создать два произвольных файла.
echo "Hello" > new1
echo "Hi" > new2

 - первому файлу присвоить права на чтение, запись для владельца и группы; на чтение для всех.
chmod 664 new1
chmod u+rw-x,g+rw-x,o+r-wx new1 

 - второму файлу присвоить права на чтение, запись только для владельца. Сделать это в численном и символьном виде.
chmod 644 new2
chmod u+rw-x,g+r-wx,o+r-wx new2

 4. Создать пользователя, обладающего возможностью выполнять действия от имени суперпользователя.
 
sudo useradd -ou 0 -g 0 superuser 
sudo passwd superuser 
grep superuser /etc/passwd 

superuser:x:0:0::/home/superuser:/bin/sh

 5. * Создать группу developer, несколько пользователей, входящих в эту группу. 
 
sudo groupadd developer sudo useradd -m -G developer -s /bin/bash user1 
sudo useradd -m -G developer -s /bin/bash user2 

Создать директорию для совместной работы.

mkdir develop
sudo chgrp developer ./develop

Сделать так, чтобы созданные одними пользователями файлы могли изменять другие пользователи этой группы.
chmod 775 ./develop

 6. * Создать в директории для совместной работы поддиректорию для обмена файлами, но чтобы удалять файлы могли только их создатели.
 
mkdir develop/exchange
cd develop
sudo chgrp developer ./exchange
chmod 1775 ./exchange

 7. * Создать директорию, в которой есть несколько файлов. 
 
mkdir invisible
cd invisible
echo Hello! > text1
echo Hello world! > text2
echo Hello world and people! > text3

Сделать так, чтобы открыть файлы можно только, зная имя файла, а через ls список файлов посмотреть нельзя.

chmod -r invisible

Например:
anastasia@anastasia-VirtualBox:~$ ls -l ./invisible
ls: cannot open directory './invisible': Permission denied
anastasia@anastasia-VirtualBox:~$ ls -l ./invisible/text1
-rw-rw-r-- 1 anastasia anastasia 7 Apr  2 12:32 ./invisible/text1
